The Shell

Let's talk about the shell, as previously mentioned in the previous terminal section, a shell is a program that interprets simple commands into complex instructions. There are many shells out there, but the one I am going to talk about is Bash. As far as my understanding goes, there isn't much difference between the shells. Some have some features that others don't, but they all essentially accomplish the same job.

Let's open up a terminal, shall we?

When I first open up the terminal get greeted with a prompt like this:

michael@Michael-Server:~$

In this prompt I can type anything.

michael@Michael-Server:~$ uhnnfjninfinfijneff
bash: uhnnfjninfinfijneff: command not found...
michael@Michael-Server:~$

apparently uhnnfjninfinfijneff isn't a command. Let's try another

michael@Michael-Server:~$ pwd
/home/michael
michael@Michael-Server:~$

huh. Something came back. Turns out `pwd` is an actual command, it is in fact the abbreviation of 'Print Working Directory'. Well, how does the shell know `pwd` is a command and `uhnnfjninfinfijneff` isn't? When you enter a command, the shell looks for the command in a series of directories in the computer. Once it finds the command, it executes it. If you are familiar with Windows, you probably know how to open up your web browser. The long way of doing it is by opening up file explorer and searching for the directory for the browser and then double clicking on the browser executable. When you double click the executable, you essentially execute, the executable. Man, I got to stop using that word lol. When you enter a command, it does the same. It executes the executable. Ok, but whereis exactly `pwd`? how did the shell find it? As I said, the shell looks in a series of directories to find it. Which directories? The directories the shell looks in are set in the PATH environment variable. The what now? Just keep reading...

michael@Michael-Server:~$ echo $PATH
/usr/share/Modules/bin:/usr/local/bin:/usr/bin:/usr/local/sbin:/usr/sbin
michael@Michael-Server:~$

What did I just do? `echo` is another command. It just prints stuff out. If you are familiar with C, it's basically shell's printf. What is `$PATH`? It is variable. I could set the variable to something else if I wanted. I could say:

michael@Michael-Server:~$ PATH=<something else>

but this variable is special. Because it is special it is called an environment variable. Ok, there is an actual reason why it is called an environment variable. When the shell first loads up, it starts a list of environment variables. One of them is the PATH variable. To print a variable using echo you just simple prepend the variable name with dollar sign, which is why ` echo $PATH ` prints the content of PATH. The directories are separated by a colon. To see a list of all the enviroment variables simply run `env`.

Now, going back to `pwd`, whereis it? Let's look in the directories. To do that we run the command `ls`.

michael@Michael-Server:~$ echo $PATH
/usr/share/Modules/bin:/usr/local/bin:/usr/bin:/usr/local/sbin:/usr/sbin
michael@Michael-Server:~$ ls /usr/share/Modules/bin
add.modules  createmodule.py  createmodule.sh  mkroot
michael@Michael-Server:~$ ls /usr/local/bin
michael@Michael-Server:~$ ls /usr/bin

wow... this is a huge list. I actually shorted it. We can still find it. Let's just add pwd at the end of the of it, and see what `ls` comes back with

michael@Michael-Server:~$ ls /usr/bin/pwd
/usr/bin/pwd
michael@Michael-Server:~$

wow, but wait, how do we know this is correct? How do we know ls isnt lying? Lets do:

michael@Michael-Server:~$ ls /usr/bin/uhnnfjninfinfijneff
ls: cannot access '/usr/bin/uhnnfjninfinfijneff': No such file or directory
michael@Michael-Server:~$

ok `ls` is telling the truth. I wonder if there is an easier way to find the command? Wait, there is! The command is `whereis` ;)

michael@Michael-Server:~$ whereis pwd
pwd: /usr/bin/pwd /usr/include/pwd.h /usr/share/man/man1p/pwd.1p.gz /usr/share/man/man1/pwd.1.gz
michael@Michael-Server:~$
